# 개발일지

## 2026-06-08 (3) — 배포 방식을 main/docs 로 전환

### 오늘 한 일
- GitHub Pages가 `gh-pages` 브랜치가 아닌 `main` 브랜치 소스(raw `index.html`)를 그대로
  서빙하고 있어, 빌드된 자산(JS/CSS/favicon)이 모두 404로 응답되는 문제를 발견
- 저장소를 `main` 브랜치 하나로 단순하게 운영하기 위해 배포 방식을 다음과 같이 변경
  - `vite.config.js`에 `build.outDir: 'docs'` 설정 → 빌드 결과물이 저장소 루트의 `docs/`에 생성
  - `public/.nojekyll` 추가 (GitHub Pages의 Jekyll 처리로 인한 `assets/` 폴더 누락 방지)
  - 더 이상 쓰지 않는 `gh-pages` 패키지와 `predeploy`/`deploy` 스크립트 제거
  - 기존 `docs/개발일지.md`는 빌드 출력 폴더와 경로가 겹치므로 저장소 루트의
    [개발일지.md](개발일지.md)로 이동
- README의 빌드/배포 안내를 `npm run build` → `docs/` 커밋·푸시 흐름으로 갱신

### 안내 (사용자 액션 필요)
- GitHub 저장소 **Settings → Pages → Build and deployment → Source**를
  `Deploy from a branch` → Branch `main` / Folder `/docs` 로 한 번 설정해야 합니다.
  (이전에 만든 `gh-pages` 브랜치는 더 이상 사용하지 않으므로 그대로 두거나 삭제해도 무방합니다.)

## 2026-06-08 — 프로젝트 셋업 & 1차 페이지 구축

### 오늘 한 일

**1. 개발 환경 구축**
- 저장소(`Cjy0913/rest-2-`)를 기준으로 Vite + React 19 프로젝트를 스캐폴딩
- 라우팅을 위해 `react-router-dom` 추가
- 패키지명을 `codeblue-systems-site`로 변경, `index.html` 타이틀/메타 설명을 회사 소개에 맞게 수정

**2. 디자인 컨셉 정리**
- 회사 컨셉: IT 개발 회사 "코드블루시스템즈" (CodeBlue Systems), 2003년 9월 23일 설립
- 사업 영역: 컴퓨터 판매 / 프로그램 개발 / 웹 개발
- 컬러 가이드를 `src/index.css`의 CSS 변수로 정의
  - 메인: 다크블루 `#0b1e3d`, 로열블루 `#3b5bdb`
  - 포인트: 다크그린 `#1b5e3f`, 다크레드 `#7a2530`
- 위 4색을 헤더/히어로(다크블루 그라데이션), 버튼·강조 텍스트(로열블루), 사업 영역 카드 포인트(그린/레드)에 일관되게 적용

**3. 공통 레이아웃 컴포넌트**
- `Header` — 반응형 네비게이션(모바일 햄버거 메뉴 포함), 로고 + 회사명/설립연도 표기
- `Footer` — 회사 정보, 바로가기, 사업영역, 오시는 길 4단 구성
- `Layout` — `react-router-dom`의 `Outlet`으로 헤더/본문/푸터를 감싸는 공통 레이아웃

**4. 페이지 4종 제작 (`react-router-dom` 라우팅)**
- `/` 홈 — 히어로, 핵심 통계(설립연도·경력·사업영역 수·누적 프로젝트), 3대 사업 카드, 선택 이유, CTA 배너
- `/about` 회사소개 — 인사말, 회사 개요표, 핵심가치(신뢰·전문성·동행), 2003년부터의 연혁 타임라인
- `/services` 사업영역 — 컴퓨터 판매·프로그램 개발·웹 개발 각 영역의 상세 설명과 제공 항목, 진행 절차(상담→제안→구축→운영) 4단계
- `/contact` 문의하기 — 연락처/오시는 길 정보, 문의 유형 선택이 가능한 입력 폼(프론트엔드 상태로 제출 처리)

**5. 검증**
- `npm run dev`로 로컬 구동 후 Playwright 헤드리스 브라우저로 4개 페이지 전체 스크린샷 캡처
- 콘솔/페이지 에러 없음 확인, 헤더 내비게이션 클릭 시 라우팅 정상 동작 확인
